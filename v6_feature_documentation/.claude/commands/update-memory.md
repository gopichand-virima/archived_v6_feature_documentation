# /update-memory — Project Memory Updater

**Invoke with:** `/update-memory`

**Purpose:** One-command entry point to keep the project's AI memory system current. Detects what changed, updates only the affected sections, and bumps the version. Works by routing to the appropriate depth of memory tooling.

---

## Quick decision: which depth do you need?

| Situation | Action |
|-----------|--------|
| Values changed, paths moved, CI contracts updated | Use **this command** (routes to `/sync-repo-memory`) |
| New top-level section needed, schema changed, routing rules changed | Use `/memory-maintainer` directly |
| Major restructure or breaking schema change | Use `/memory-maintainer` + bump major version manually |

---

## Step 1 — Run drift detector

```bash
pnpm memory:check
```

| Exit | Meaning | Do this |
|------|---------|---------|
| `0` GREEN | Memory is current | Nothing needed — confirm with `pnpm validate` and stop |
| `1` AMBER | File changes detected — memory may be stale | Continue to Step 2 |
| `2` RED | Value mismatch — memory records wrong facts | Continue to Step 2 immediately |

---

## Step 2 — Identify what changed

Read the drift report output. It lists:
- Which **categories** are flagged (e.g., "CI/CD workflow changes", "Base path / build config")
- Which **files** changed in those categories
- Which **memory sections** need updating

Use the routing table from `sync-repo-memory` or run:

```bash
node -e "const m=JSON.parse(require('fs').readFileSync('docs/memory/repo-memory.json','utf8')); console.log(Object.keys(m).filter(k=>!k.startsWith('\$') && !['memory_version','generated_at','maintained_by'].includes(k)).join('\n'));"
```

To see all current memory sections.

---

## Step 3 — Update the affected sections

Edit `docs/memory/repo-memory.json` directly. For each flagged section:

1. Read the actual source (the changed file)
2. Compare it with the current memory value
3. Update only what is now inaccurate or missing

**Key routing shortcuts:**

| What changed | Memory section to update |
|---|---|
| Vite base path, build output | `repo_contracts` |
| Deployment model, CI workflows | `ci_cd_contracts` |
| Live URL | `project_identity.live_url` |
| Navigation sidebar behavior | `navigation_contract` |
| Right TOC behavior | `toc_contract` |
| Theme tokens, dark/light mode | `theme_and_branding`, `dark_mode_contrast` |
| Homepage cards | `homepage_cards` |
| Search behavior | `search_capabilities` |
| Version dropdown | `versioning_and_dropdown_behavior` |
| Breadcrumbs, URL structure | `url_management` |
| Generation pipeline stages | `generation_pipeline` |
| Module index.md contract | `publishing_contract` |
| Content rendering (MDX/sanitize) | `content_rendering_rules` |
| Security headers, scanning | `security_contract` |
| Branch naming | `branch_naming` |
| New anti-pattern discovered | `anti_patterns_to_avoid` — **append only, never delete** |

---

## Step 4 — Also update supporting artifacts if needed

| If this changed | Also update this |
|---|---|
| Generation pipeline step order | `.claude/commands/repo-maintainer.md` — Step 3 rules |
| New slash command added | `.claude/commands/repo-maintainer.md` — Quick Reference table |
| New validation step added | `repo-memory.json` § `validation_gates` |
| Memory system itself changed | `docs/memory/README.md` |

Do **not** duplicate facts into `CLAUDE.md` or `README.md` — `repo-memory.json` is the single machine-readable source.

---

## Step 5 — Bump version and validate

```bash
pnpm memory:update        # patch-bumps memory_version + sets generated_at + validates JSON
pnpm memory:check         # must return GREEN (exit 0) before committing
```

Version guidance:
- **Patch** (`x.x.+1`): Updated values, added facts — `pnpm memory:update` handles this
- **Minor** (`x.+1.0`): New top-level section — edit version manually first, then `pnpm memory:update --validate`
- **Major** (`+1.0.0`): Breaking schema change — use `/memory-maintainer`

---

## Step 6 — Commit

Include memory changes in the session commit:

```bash
git add docs/memory/repo-memory.json
# Add any other updated artifacts (repo-maintainer.md, README.md, etc.)
git commit -m "chore(memory): sync repo-memory — <what changed> — v{new_version}"
```

---

## What NOT to update

Skip memory updates for:
- Trivial refactors (variable renames, comment typos)
- Changes to `src/pages/content/` (pipeline-generated — tracked by manifests)
- Changes to `src/data/navigationData.ts` / `src/utils/indexContentMap.ts` (auto-generated by sync-toc)
- Image asset additions
- Lockfile (`pnpm-lock.yaml`) changes
- Test-only changes in `scripts/__tests__/`

---

## Escalate to `/memory-maintainer` when

- You need to add a **new top-level section** to `repo-memory.json`
- You need to update the **JSON Schema** (`repo-memory.schema.json`)
- You need to add/update **routing rules** in `scripts/memory/change-routing-map.ts`
- `pnpm memory:check` returns RED for a **value mismatch** (not just AMBER file-change)
- You need to do a full **memory health review** across all sections
