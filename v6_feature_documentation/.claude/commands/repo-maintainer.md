# Repo Maintainer ‚Äî v6 Feature Documentation

**Invoke with:** `/repo-maintainer`

**Purpose:** Load the canonical AI memory index and apply all repo contracts, rules, and anti-patterns for the current session. Run this at the start of any session involving architecture, content, theme, layout, or infrastructure changes.

---

## Step 1 ‚Äî Load Memory

Read the canonical memory file:

```
docs/memory/repo-memory.json
```

Apply everything in it as immutable session rules. Do not skip sections. Pay particular attention to:

- `repo_contracts` ‚Äî pnpm, TypeScript-only, dist/ output, base: './' (relative, GitHub Pages root-served)
- `anti_patterns_to_avoid` ‚Äî common mistakes and correct alternatives
- `tooling_and_build.tailwind_v4_critical_notes` ‚Äî pre-compiled CSS gotchas
- `layout_contracts` ‚Äî two-column reading layout, back-filled utilities
- `content_pipeline` ‚Äî TOC-driven sync-toc ‚Üí doc-graph ‚Üí search-index pipeline
- `publishing_contract` ‚Äî module index.md ownership and format
- `validation_gates` ‚Äî all gates must pass before merge
- `branch_naming` ‚Äî pattern and separator rules

---

## Step 2 ‚Äî Confirm Context

After loading, confirm:

1. What branch is currently active: `git -C /c/github/v6_feature_documentation branch --show-current`
2. What is uncommitted: `git -C /c/github/v6_feature_documentation status --short`
3. Whether validation is passing: run `pnpm typecheck` if any source changes are staged

---

## Step 2.5 ‚Äî Check Memory Drift

Run the drift detector to confirm memory is current before starting work:

```bash
pnpm memory:check
```

**Exit codes:**
- `0` (GREEN) ‚Äî memory is current, proceed
- `1` (AMBER) ‚Äî file changes detected since last memory update; review flagged sections
- `2` (RED) ‚Äî value mismatch detected; memory records wrong facts ‚Äî fix before proceeding

If AMBER or RED, run `/sync-repo-memory` to resolve drift before making new changes. This ensures you are working from accurate contract information.

---

## Step 3 ‚Äî Apply Session Rules

For the duration of this session:

### Never do these (from anti_patterns_to_avoid)
- Use `npm` or `yarn` ‚Äî always `pnpm`
- Hand-edit `src/data/navigationData.ts` or `src/utils/indexContentMap.ts` ‚Äî auto-generated by `pnpm sync-toc`
- Hand-edit `public/doc-graph.json` or `public/search-index.json` ‚Äî auto-generated by prebuild
- Set `outDir` to `build/` ‚Äî must be `dist/`
- Change the Vite base path ‚Äî must remain `'./'` (relative; GitHub Pages serves from root)
- Use arbitrary Tailwind v4 classes not confirmed to be in `src/index.css` ‚Äî back-fill in `globals.css` instead
- Add `.js` source files under `src/` ‚Äî TypeScript only
- Write "Virima" ‚Äî always "VIRIMA" in brand text
- Create branches with `/` separators ‚Äî use `-` instead (e.g., `fix-layout-bug-6_1-31032026`)
- Commit to `main` without passing `pnpm validate` and `pnpm build`
- Use `npm run` instead of `pnpm run`

### Always do these
- Run `pnpm typecheck` before committing any `.ts` / `.tsx` change
- Run `pnpm validate` before merging any branch
- Use `pnpm build` to confirm no build errors
- When adding new utility classes, check `src/index.css` first; if not present, add to `src/styles/globals.css`
- Use named Tailwind v4 tokens (`max-w-7xl`, `w-64`) not arbitrary values (`max-w-[1280px]`, `w-[256px]`)
- Check `docs/memory/repo-memory.json` before restructuring any folder or changing any contract

## Step 4 ‚Äî Update Memory When Contracts Change

If this session changes any architecture, contract, or functional spec, update `docs/memory/repo-memory.json`:

1. Apply targeted edits to the relevant section(s) ‚Äî use routing map to identify which sections
2. Run `pnpm memory:update` ‚Äî automatically bumps `memory_version` patch and `generated_at`
3. Run `pnpm memory:check` ‚Äî confirm GREEN before committing
4. Include the memory update in the session commit

Or use the guided skill: `/sync-repo-memory`
