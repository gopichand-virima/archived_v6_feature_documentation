## 4. The real gap to implement

**GREEN gate must always create branch + PR (never commit directly to main)**

### Current behaviour
```
GREEN  → git push directly to main        ← must change
AMBER  → branch + PR                      ← correct
RED    → branch + PR + fail workflow      ← correct
```

### Target behaviour
```
GREEN  → branch + PR (labelled ✅ safe to merge)
AMBER  → branch + PR (labelled ⚠️ review required)
RED    → branch + PR + fail workflow (labelled 🔴 blocking)
```

### Design decision: Option A — unified commit step
Replace the two separate commit steps (`GREEN: Commit to main` and `AMBER/RED: Create branch + PR`) with a single `Create branch + PR` step that always runs when `has_generated == true`. Gate result drives only PR title, label, and failure exit code.

### Branch naming (enterprise policy)
```
autofeature-featuredocs-6_1-{DDMMYYYY}
```
- No forward slashes
- Flat naming only
- Date format: DDMMYYYY

### PR bodies by gate
- **GREEN:** "✅ All validation checks passed. Safe to review and merge."
- **AMBER:** "⚠️ Warnings detected — reviewer judgment required before merging."
- **RED:** "🔴 Blocking issues detected — fix required before merge. Workflow has failed."
