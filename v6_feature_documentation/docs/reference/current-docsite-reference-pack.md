# 8. External Settings and Permissions Required

These settings live outside the repository and must be configured manually in GitHub. They cannot be configured via committed files. This is a required checklist for any new repo setup.

**1. GitHub Pages source (required for deployment)**
- Location: `Settings → Pages → Source`
- Required value: **GitHub Actions** (NOT branch-based)
- Why: `deploy-pages.yml` uses `actions/upload-pages-artifact` and `actions/deploy-pages` — these require Actions-based source. Branch-based source will cause deployment to fail.

**2. GitHub Actions PR creation permission (required for pipeline PRs)**
- Location: `Settings → Actions → General → Workflow permissions`
- Required value: **"Allow GitHub Actions to create and approve pull requests"** checkbox must be checked
- Why: `generate-v6feature-docs.yml` calls `gh pr create` using `${{ github.token }}`. Without this permission, PR creation fails with a 403. The branch is still pushed, but the PR must be created manually.

**3. Repository secret: ANTHROPIC_API_KEY**
- Location: `Settings → Secrets and variables → Actions → New repository secret`
- Value: Anthropic API key (starts with `sk-ant-...`)
- Why: Used by `generate-feature-doc.ts` to call the Anthropic API for PRD classification and document generation.
- Security note: Scope this key to specific domains at the Anthropic console. Apply strict usage limits.

**4. Repository secret: PRD_REPO_TOKEN**
- Location: `Settings → Secrets and variables → Actions → New repository secret`
- Value: GitHub PAT (classic or fine-grained) with `contents: read` on the `v6_prds` repo
- Why: `generate-v6feature-docs.yml` checks out the `v6_prds` repo at `_prd-source/` using this token.
- Security note: Use a fine-grained PAT scoped to the specific `v6_prds` repo with minimum permissions.

**5. PRD repo trigger workflow**
- Location: `virima-products/v6_prds/.github/workflows/`
- Required: A workflow that fires `repository_dispatch` to `v6_feature_documentation` on PRD merges to `main`
- Payload must include: `{ changes: [...], source_commit, source_author, source_ref, full_rebuild }`
- Without this: Generation pipeline only runs on `workflow_dispatch` (manual trigger)

**Summary Table:**

| Setting | Location | Required Value |
|---------|---------|---------------|
| Pages source | Settings → Pages → Source | GitHub Actions |
| PR creation | Settings → Actions → General | Allow GitHub Actions to create and approve pull requests ✓ |
| ANTHROPIC_API_KEY | Settings → Secrets → Actions | `sk-ant-...` key |
| PRD_REPO_TOKEN | Settings → Secrets → Actions | PAT with read access to PRD repo |
| PRD repo trigger | v6_prds repo workflows | `repository_dispatch` to this repo |

**What to ask:**

- Are there missing accessibility, search, or BFF capabilities blocking production readiness?
